# LQAY Style Blog Redesign Spec

**Date:** 2026-06-03

**Goal**

在保留 `Hexo + Argon` 主题数据流、分页、文章渲染与评论接入能力的前提下，将站点改造成接近 `blog.lqay.cn` 的完整版本：首页、文章页、悬浮播放器、看板娘、评论区、搜索入口和移动端交互统一到同一套视觉与结构系统中。

**Current Project Context**

- 站点框架是 Hexo，主题为 `themes/argon`。
- 主题关键入口集中在 `layout.ejs`、`header.ejs`、`sidebar.ejs`、`index.ejs`、`post.ejs`、`footer.ejs` 及若干 `_partial` 模板。
- 主题已支持自定义导航、Banner、页脚、评论系统和脚本注入。
- 现有测试 `tests/layout-shell.test.mjs` 已经期望出现 `lq-shell`、`lq-left-column`、`lq-right-column` 等新结构，但当前生成页面仍是默认 Argon 壳层。
- 当前配置和生成 HTML 中存在中文乱码痕迹，说明编码或文案源需要先清理。

**Reference Interpretation**

本次设计基于用户提供的目标站首页截图，而不是直接抓取目标站源码。可以明确提炼出的设计语言如下：

- 首页为全屏沉浸式首屏，主视觉依赖大背景图与浅色遮罩。
- 顶部导航透明悬浮，图标与文字并列，视觉权重较轻。
- 首页正文首屏只保留站点标题与一句副标题，信息密度低。
- 左下是看板娘类浮动挂件，右下是悬浮音乐播放器。
- 页面整体是“轻、淡、二次元、悬浮感”，而不是 Argon 默认的内容优先布局。
- 站点应具备明显的“首页特化”外壳，但文章页也需要延续同一套视觉系统。

**Architecture**

本次改造不新建主题，而是在 `argon` 主题内部做“保留引擎，重写壳层”的定制：

1. 保留 Argon 的内容生成、分页、文章 helper、评论接入和现有脚本能力。
2. 以 `lq-shell` 为新页面骨架，统一首页、文章页和独立页的外层结构。
3. 首页使用新的沉浸式 Hero 与文章流组合结构。
4. 右下播放器、左下看板娘、评论区等作为可配置功能模块挂载到壳层中。
5. 样式层不继续堆叠零散覆盖，而是建立一层新的 LQ 风格布局与组件命名。

**Page Model**

**1. Home**

- 顶部透明导航，支持图标、文字、展开菜单和搜索按钮。
- 首屏 Hero 全屏展示背景图、站名、slogan、滚动提示。
- 首屏下方进入文章流，可保留分页。
- 首页右侧可放站点信息、推荐标签、最近文章、公告等辅助块。
- 首页应支持播放器和看板娘悬浮，且不遮挡主要标题区域。

**2. Post**

- 保留文章正文的可读性，避免首页那种过度空旷。
- 文章页外层仍使用统一 `lq-shell`。
- 主栏包含文章标题、封面图、元信息、正文、标签、上下篇、评论区。
- 右栏展示目录、站点卡片、最近文章或分类入口。
- 背景与导航视觉与首页保持连续，但正文卡片层级更明确。

**3. Standalone Page**

- 关于页、友链页等仍走统一壳层。
- 内容区域使用与文章页相近的卡片系统，但去掉文章特定模块。

**Component Breakdown**

**1. Navigation**

- 使用透明导航覆盖在首页首屏顶部。
- 支持图标 + 文本导航项。
- 首页滚动后可以切换为更高可读性的半透明或实色背景。
- 移动端使用折叠菜单，不保留 Argon 当前笨重的默认展开感。

**2. Hero**

- 以大背景图和轻遮罩为主。
- 中央标题区包含站点标题与一句副标题。
- 允许后续接打字效果，但首版不依赖复杂动画。
- 提供向下滚动提示。

**3. Content Shell**

- 页面整体引入 `lq-shell`。
- 建议桌面端为三段式：左栏、主栏、右栏。
- 移动端压缩为单列，左右功能区折叠为顶部入口或底部弹层。

**4. Post Card System**

- 首页卡片改为新风格 `lq-article-card`。
- 独立页卡片使用 `lq-page-card`。
- 文章元信息、标签、摘要、缩略图统一新层级与命名。

**5. Floating Widgets**

- 右下悬浮播放器独立模块化。
- 左下看板娘独立模块化。
- 回到顶部、搜索、阅读进度、侧栏切换等也应统一为悬浮功能层，而不是与 Argon 默认按钮风格混杂。

**6. Comment System**

- 评论模块保留 Argon 的现有接入方式，优先使用 `giscus` 或 `waline`。
- 评论区需要嵌入新的卡片视觉，不直接裸露默认样式。

**7. Search**

- 保留现有本地搜索数据源 `search.xml`。
- 搜索入口样式要融入导航或壳层，不保留纯 Bootstrap 默认模态视觉。

**Configuration Strategy**

配置分三层：

1. `_config.yml`
   用于站点级标题、URL、生成器和插件开关。
2. `source/_data/argon.yml`
   用于主题级视觉与功能配置，是本次改造的主要配置入口。
3. 自定义新增配置
   建议增加一组 `lq_*` 或 `custom_*` 字段，管理首页背景、看板娘、播放器、导航图标、右栏模块开关等。

**Asset Strategy**

- 首页背景图、站点 Logo、看板娘立绘、播放器封面等素材统一放在 `source/img` 或主题静态资源目录中。
- 对于播放器，如果需要自定义封面、歌单或单曲信息，应通过配置读取，避免硬编码在模板里。

**Error Handling and Risk Control**

- 先处理中文乱码和配置编码问题，否则后续导航、标题和说明文字都会被污染。
- 不直接改 Argon 合并后的压缩资源，优先改模板与 `style.css`。
- 尽量把新增样式和结构命名放到 `lq-*` 前缀，避免和 Argon 原类冲突。
- 播放器与看板娘都是“易破版”的浮动组件，需要预留移动端降级策略。
- 评论区外链脚本可能影响首屏加载，建议其位置放在正文下方，不阻塞首页首屏渲染。

**Testing Strategy**

- 更新并通过 `tests/layout-shell.test.mjs`，确保首页、文章页、独立页生成新的壳层结构。
- 增加至少以下验证：
  - 首页包含 `lq-shell`、`lq-hero`、`lq-search-trigger`。
  - 文章页包含 `lq-article-card`、评论容器、右栏容器。
  - 独立页包含 `lq-page-card`。
  - 配置关闭播放器或看板娘时，页面不渲染对应模块。
- 每次模板改动后都需要重新 `hexo generate` 验证生成结果。

**Out of Scope for First Pass**

- 不在第一轮实现复杂的实时背景特效。
- 不在第一轮做多主题切换系统。
- 不在第一轮完全复制目标站每一个小装饰，而是优先完成结构、氛围和核心功能对齐。

**Acceptance Criteria**

- 首页具备接近目标站的沉浸式视觉与透明导航。
- 首页、文章页、独立页都切换到统一 `lq-shell` 结构。
- 页面存在完整的悬浮播放器、看板娘和评论区集成能力。
- 移动端不会因为浮动组件造成主内容不可读。
- 测试与静态生成结果一致，不再出现“测试期望新壳层但页面仍是旧模板”的状态。
