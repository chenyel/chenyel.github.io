# 右下角浮动播放器 Signal HUD 改造设计

## 背景

右下角浮动播放器已经恢复可用，并完成了收起态把手式交互调整。但当前视觉仍偏普通白色卡片，和站内已经建立起来的 `signal layer` 语言存在明显落差：

- 播放器功能可用，但第一眼记忆点不足。
- 卡片质感、按钮层级、进度条表现都偏常规，不像站内其他高完成度模块。
- 站点已经有 `lq-divergence-loader` 这套较明确的科技感视觉语言，播放器适合借用其气质，但不能直接复制。

本次设计目标是在不改变播放器体量、位置和基础交互的前提下，把它升级为一个中等强度的 `Signal HUD` 小型面板。

## 目标

- 保留右下角播放器当前结构、尺寸级别和可用性。
- 让播放器视觉上更接近站内 `signal layer` 气质。
- 提升“第一眼记忆点”，但不压过正文和主要内容区。
- 维持收起态“一小条卡片边 + 箭头”的当前交互。
- 不引入新的复杂依赖或新的播放数据链路。

## 非目标

- 不改成全新交互范式，例如纯 hover 展开或全自动弹出。
- 不重做播放器 DOM 结构为完全不同的组件。
- 不把视觉做成高频闪烁或重型赛博 HUD。
- 不改变现有播放逻辑、歌单来源和页面级显示规则。

## 设计定稿

### 1. 整体方向

视觉方向确定为 `Signal HUD`：

- 使用站点主题色作为主基底，不脱离整站色彩体系。
- 在主题色之上叠加较克制的冷色 signal 光，而不是直接改成纯青蓝面板。
- 强化“这是一块被唤醒的小型信号面板”的感觉，而不是“一个普通音乐卡片换了皮肤”。

整体科技感强度控制在中等：

- 明显区别于普通卡片。
- 与 `lq-divergence-loader` 有亲缘关系。
- 仍然保持博客组件的分寸感，不抢正文阅读焦点。

### 2. 外观

播放器保留现有右下角位置和大致体量，不做放大或重新排布。外观升级重点如下：

- 面板主体保留半透明基底，但从纯白玻璃改为更偏“信号面板”的分层玻璃。
- 外框增加细描边与弱发光边缘，形成精密面板轮廓。
- 背景内部加入极淡的信号纹理，例如轻网格、切角高光或局部扫描层次，但强度必须低。
- 封面区从普通方形图片区升级为“模块仓”视觉，强调它是播放器左侧的核心信息仓。
- 播放键成为视觉核心按钮，优先级高于前后切换键。
- 进度条升级为“能量槽”风格，让时间推进更有设备感。
- 状态文案、时间信息、索引信息统一成更干净的 HUD 文本层级，不使用夸张字体或过重装饰。

### 3. 配色策略

配色方案确定为：

- 保留站点主题色为底。
- 在边缘高光、能量槽、焦点按钮、局部阴影中叠加少量冷色 signal 光。
- 补充少量白银灰作为面板中性层，避免整块卡片过蓝。

这样可以同时满足两点：

- 和主站已有主题融合，不像外来组件。
- 保留科技感和辨识度，不会被误认成普通主题卡片。

### 4. 动效

播放器动效采用“被唤醒的 HUD 面板”思路，但整体保持克制：

- 展开和收起继续沿用当前整体滑出逻辑，增加更顺滑的面板唤醒感。
- 收起态露出的边缘维持当前方案，并保留少量信号感高光，让收起时也像组件的一部分。
- 静止状态只保留微弱呼吸感和冷光浮动，不做持续强动画。
- 播放时，进度条增加低强度的能量流动感。
- 按钮 hover 和激活反馈更明确，但不使用剧烈缩放或频闪。
- 封面区可加入极轻的高光扫过或仓体反射感，但频率必须低。

### 5. 收起态规则

当前已经确认的交互约束继续保留：

- 箭头按钮继续作为播放器整体的一部分存在。
- 收起后仅露出一小条卡片边与箭头，作为把手。
- 箭头与整个播放器一起联动缩进和缩放，不允许再次变成独立悬浮按钮。

### 6. 风险边界

为了保证视觉升级不破坏页面平衡，实施时必须遵守以下边界：

- 不引入强频率扫描线、频闪、连续大范围扫光。
- 不把发光强度做成比正文区域更抢眼的悬浮主视觉。
- 不把播放器颜色做成脱离主站主题的纯霓虹组件。
- 不新增依赖 hover 才能使用的关键交互。
- 不为了“科技感”牺牲按钮识别度、时间可读性和移动端可触达性。

## 实施建议

实现应尽量以样式升级为主，结构微调为辅：

- 优先修改 `themes/argon/source/style.css` 中 `lq-player` 相关样式。
- 仅在确有必要时微调 `themes/argon/layout/_partial/lq-player.ejs` 的局部结构或状态类。
- 现有 `argontheme.js` 播放逻辑不应因视觉改造而扩散重构。
- 继续使用回归测试约束播放器壳子、箭头把手和收起态关键特征。

## 验收标准

- 首页右下角播放器第一眼明显更有识别度。
- 整体气质与 `signal layer` 语言一致，但不会喧宾夺主。
- 收起态仍然易发现、易点击，展开收起体验连续。
- 文本、时间、进度、按钮在桌面和移动端都保持清晰可用。
- 不影响现有播放链路和已通过的播放器相关测试。
