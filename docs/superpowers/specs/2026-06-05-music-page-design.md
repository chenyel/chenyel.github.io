# /music 页面改造设计

## 背景

当前 `/music` 页面存在四个核心问题：

1. 页面同时出现页内音乐模块和右下角全局播放器，视觉与交互冲突明显。
2. 音乐页沿用普通内容页壳子，左右状态栏挤压主内容，双区块不够舒展。
3. 页面文案中存在“代理”“接入前”“兜底”“fallback”“coming soon”等过渡态表达，完成度不够。
4. 背景、歌词动效和播放器卡片质感偏弱，与用户提供的参考气质有差距。

本次改造目标是在不破坏其他页面的前提下，将 `/music` 调整为一个独立、统一、可直接使用的沉浸式音乐页。

## 目标

- 让 `/music` 页面只保留一个明确的主播放体验。
- 将页面改造成无两侧状态栏的双区块沉浸布局。
- 提升背景氛围、播放器质感和歌词动效表现。
- 去掉所有代理和占位性质的描述，统一成已完成的音乐页语气。
- 仅在 `/music` 页面隐藏右下角全局播放器，其他页面继续显示并可用。

## 非目标

- 不重做整站页面壳子。
- 不替换现有歌单来源。
- 不把全局播放器从全站移除。
- 不额外引入新的远端音乐服务或新的数据接口方案。

## 现状摘要

- 音乐页由 `themes/argon/layout/_partial/content-page.ejs` 中的 `post.lq_music_page` 分支渲染。
- 页面当前包含一个上方“沉浸播放器”区域、一个歌单卡片区，以及下方的 Meting/APlayer 播放器区。
- 右下角常驻播放器来自全局注入配置，不属于当前音乐页主体模块。
- 页面右栏通过 `themes/argon/layout/_partial/lq-rightbar.ejs` 在通用布局中统一挂载。
- 现有脚本 `themes/argon/source/argontheme.js` 已包含音乐页歌单、歌词、播放状态联动逻辑，可在其基础上增强，而非推倒重做。

## 设计方案

### 1. 页面布局

`/music` 页面将从普通内容页模式切换为音乐页专属展示模式：

- 保留站点顶部导航。
- 在 `/music` 页面隐藏左侧栏和右侧状态栏，让主内容区域独占横向空间。
- 主体采用双区块布局。

双区块布局定义如下：

- 左块为主音乐盒。这里直接承载当前可用的音乐盒和实际播放控制，作为页面唯一主播放器。
- 右块为歌词舞台。这里重点展示当前歌曲歌词，不再只是普通滚动列表。
- 双区块下方保留歌单区，用于切换歌单，但视觉层级低于上方主舞台。

页面宽度将较普通文章页更宽，使主播放器和歌词区在桌面端拥有足够留白，不再被压缩。

### 2. 页面级播放器策略

播放器冲突按页面级规则处理：

- `/music` 页面隐藏右下角全局播放器。
- 非 `/music` 页面保持右下角全局播放器正常显示。
- `/music` 页面内部仅保留一个明确的主播放器区域，不再出现“展示播放器”和“真正播放器”割裂的问题。

实现上优先采用页面条件类名或页面条件渲染，不影响其他页面的现有播放器启用状态。

### 3. 视觉风格

整体风格设定为“通透、柔和、分层明确的音乐舞台”，避免普通白底卡片页的平铺感。

视觉方向：

- 背景底层采用奶油粉蓝渐变。
- 中层加入缓慢漂移的 aurora 光斑。
- 表层加入非常轻的颗粒雾感或微弱高光，增强空气感。
- 左侧主音乐盒采用偏深色玻璃卡片，和浅色背景形成层级对比。
- 右侧歌词舞台保持更克制、更纯净的视觉焦点，避免过度复杂的装饰抢夺歌词本身。

### 4. 主音乐盒

主音乐盒直接承载播放体验，并提升当前组件质感：

- 强化封面展示比例、阴影和圆角。
- 优化进度条、播放按钮、前后切换按钮的视觉样式。
- 用统一的玻璃卡片语言包装播放器，而不是默认组件感。
- 保留当前可用歌单与播放逻辑，不改变用户已经可用的听歌路径。

主音乐盒的文案将从技术态表述改为音乐语境，例如：

- “正在播放”
- “歌词同步中”
- “切换到这张歌单”

不会再出现“接入前”“fallback”“兜底播放器”等描述。

### 5. 歌词舞台

右侧歌词区将从列表升级为舞台式字幕表现：

- 当前句居中、放大、提亮。
- 上一句和下一句保留，但透明度降低并轻微模糊。
- 当前句增加流光扫过效果，形成字幕舞台感。
- 切换动画采用“上浮 + 聚焦 + 光带掠过”的连续过渡，避免生硬跳变。
- 长歌词句子做自适应排版，确保长句不会挤压布局或显得凌乱。
- 无歌词时显示柔和占位文案，避免打断页面氛围。

歌词区仍保留与当前播放时间同步的核心逻辑，但表现层会更具视觉完成度。

### 6. 歌单区

歌单区保留在双区块下方，但不再承担主视觉任务：

- 歌单卡片改为更精致、整洁的卡片带。
- 点击歌单后，左侧播放器、右侧歌词、卡片选中态同步更新。
- 交互文案统一改成“播放这张歌单”“切换到这张歌单”等自然表述。

歌单区的目标是“好用但不喧宾夺主”。

### 7. 文案规范

以下类型文案全部移除或替换：

- 代理
- 接入前
- 兜底
- fallback
- coming soon
- 下一步接上
- 类似“暂时保留下面播放器”的过渡说明

替换原则：

- 全部改写为已经完成、可以直接使用的音乐页语气。
- 尽量中文化并统一风格，减少英文默认标签。
- 技术信息不直接暴露给浏览者。

### 8. 响应式策略

桌面端：

- 双区块横向展开。
- 主内容区放宽。
- 歌词舞台与主播放器形成均衡视觉。

移动端：

- 双区块降为纵向堆叠。
- 保留歌词高亮和层次感，但减少重动画。
- 缩减背景动效强度，优先保证流畅和不拥挤。

### 9. 实现边界

本次实现范围限定为：

- `/music` 页面模板与页面级布局控制。
- `/music` 页面相关样式。
- `/music` 页面相关脚本与交互文案。
- 页面级隐藏右下角全局播放器的逻辑。

不扩展为：

- 全站统一布局重构。
- 新播放器体系迁移。
- 新音乐数据源接入。

## 预期改动点

- `themes/argon/layout/layout.ejs`
  - 为 `/music` 页面增加专属壳子条件，控制左右栏显示。
- `themes/argon/layout/_partial/content-page.ejs`
  - 重构音乐页结构，去掉过渡态文案，明确双区块与主播放器关系。
- `themes/argon/layout/_partial/lq-player.ejs` 或相关页面条件逻辑
  - 在 `/music` 页面隐藏右下角全局播放器。
- `themes/argon/source/style.css`
  - 新增或重写音乐页专属布局、背景、歌词舞台与响应式样式。
- `themes/argon/source/argontheme.js`
  - 调整音乐页交互文案、歌单切换反馈、歌词舞台联动细节。
- `source/_data/argon.yml`
  - 清理或替换音乐页相关的过渡态配置文案。

## 验证标准

完成后应满足以下标准：

1. `/music` 页面不显示右下角全局播放器。
2. 离开 `/music` 页面后，右下角全局播放器恢复显示。
3. `/music` 页面不显示左右状态栏，主内容宽度明显更舒展。
4. 左侧主音乐盒可以直接播放当前音乐内容。
5. 右侧歌词区具备清晰的当前句高亮与平滑切换效果。
6. 页面中不再出现“代理”“fallback”“兜底”“coming soon”等过渡文案。
7. 桌面端双区块不拥挤，移动端布局不崩、不挤。
8. 视觉气质较当前版本有明显提升，并与整站风格保持兼容。

## 风险与取舍

- 现有主题是通用内容页壳子，音乐页专属布局可能需要页面条件分支；这比全站重构更安全，也更符合本次范围。
- 若参考 GitHub 页面中的某些效果过于依赖其专用组件或结构，本次会优先保证“整体气质更高级、体验更统一”，而不是逐像素复刻。
- 歌词特效会在桌面端表现更强，移动端做适度降级，以保证性能和清晰度。

## 结论

本次改造将 `/music` 从“普通页面里塞了多个播放器的过渡页”升级为“仅在该页隐藏全局播放器、由主音乐盒和歌词舞台共同组成的沉浸式音乐页”。页面核心是统一、舒展、好看、可直接听歌，并且不破坏其他页面现有使用习惯。
